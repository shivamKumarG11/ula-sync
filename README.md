# Traveloop

**AI-powered travel planning — build smarter itineraries, discover curated destinations, and connect with fellow travellers.**

> Built for the Odoo × KAHE Coimbatore Hackathon 2026

---

## What it does

Traveloop is a full-stack travel planning platform that combines intelligent AI assistance with a curated database of 20 global destinations. Users can plan multi-city trips, get day-by-day itineraries generated by AI, track budgets, manage packing lists, and share experiences with the community.

### Key Features

| Feature | Description |
|---|---|
| **AI Itinerary Generator** | Generate complete day-by-day plans for any trip based on your pace, interests, and budget |
| **Visual Itinerary Builder** | Drag-and-drop stops, add activities, reorder your route |
| **Budget Tracker** | Real-time cost breakdown across stops and activity categories |
| **Packing Checklist** | AI-suggested packing lists tailored to your destinations and activities |
| **Trip Notes** | Rich notes per trip with full edit history |
| **Invoice Export** | Generate a PDF expense invoice for any trip |
| **Weather Widget** | Live weather data for each stop |
| **Travel Community** | Post stories, like and comment on other travellers' experiences |
| **City Explorer** | 20 hand-curated destinations with cost data, activities, food, and shopping guides |
| **AI Chat Assistant** | Floating chatbot aware of your active trip context |

## Tech Stack

### Frontend
- **React 18** + **TypeScript 5** (strict mode)
- **Vite 5** — fast builds with `@` path alias
- **TanStack Query v5** — server state with optimistic updates
- **Zustand** — client state (auth, UI, AI chat, currency)
- **Tailwind CSS** + **shadcn/ui** (Radix primitives)
- **Framer Motion** — page transitions and animations
- **React Three Fiber** — 3D globe scene on landing page
- **React Hook Form** + **Zod** — type-safe form validation
- **@hello-pangea/dnd** — drag-and-drop itinerary reordering
- **Recharts** — budget pie charts and admin analytics

### Backend
- **Flask 3** (Python) with application factory pattern
- **PostgreSQL** + **SQLAlchemy 2** + **Alembic** migrations
- **Flask-JWT-Extended** — httpOnly cookie JWT auth with refresh token rotation
- **Flask-Limiter** — rate limiting on all public endpoints
- **Marshmallow** — request/response serialisation
- **WeasyPrint** — server-side PDF invoice generation

### AI & External APIs
- **Token Router** — LLM gateway (OpenAI-compatible) for all AI features
- **Amadeus** — real-time flight search
- **OpenTripMap** — points of interest enrichment
- **Unsplash** — auto city cover photos
- **Ticketmaster** — events at travel destinations
- **Yelp Fusion** — restaurant recommendations
- **AviationStack** — live flight status

> All external APIs degrade gracefully — the app works without any of them.

## Getting Started

### Prerequisites

- Python 3.11+
- Node.js 20+
- PostgreSQL 15+

### 1. Clone

```bash
git clone https://github.com/your-org/ula-sync.git
cd ula-sync
```

### 2. Backend setup

```bash
cd backend

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env — set SECRET_KEY, JWT_SECRET_KEY, DATABASE_URL at minimum

# Create the database
createdb traveloop

# Run migrations
flask db upgrade

# Seed city data (loads all 20 destinations from data/)
python scripts/seed.py

# Start the server
python run.py
# → Running on http://localhost:5000
```

### 3. Frontend setup

```bash
cd frontend

# Install dependencies
npm install

# Configure environment
cp .env.example .env
# VITE_API_URL=http://localhost:5000 is set by default
# Add VITE_MAPBOX_TOKEN for maps (optional)

# Start dev server
npm run dev
# → Running on http://localhost:5173
```

## API Overview

All endpoints are prefixed `/api/v1/`. Authentication uses httpOnly JWT cookies.

| Module | Prefix | Description |
|---|---|---|
| Auth | `/auth` | Register, login, logout, refresh, me |
| Users | `/users` | Profile update, saved cities |
| Trips | `/trips` | CRUD, share |
| Stops | `/trips/:id/stops` | CRUD, reorder |
| Activities | `/activities`, `/stops/:id/activities` | City catalog + stop activities |
| Budget | `/trips/:id/budget` | Aggregated cost breakdown |
| Notes | `/trips/:id/notes` | Trip notes CRUD |
| Packing | `/trips/:id/packing` | Packing items, toggle packed |
| Invoice | `/trips/:id/invoice` | CRUD + PDF export |
| Community | `/community` | Posts, comments, likes |
| AI | `/ai` | Itinerary, activities, transport, review, packing, chat |
| External | `/external` | Weather, POIs, events, flights, hotels |
| Admin | `/admin` | Users list, city management, analytics |

## Graceful Degradation

The app is designed to never break if an external service is unavailable:

- **No AI key** → all AI buttons show a friendly fallback message; manual planning still works fully
- **No Mapbox token** → map components are hidden; all other features work
- **No weather API** → weather widget is hidden on trip info page
- **No flight/hotel APIs** → transport recommendations use AI text suggestions
- **Image load failure** → local bundled city photos serve as fallback
- **Backend 401** → automatic token refresh with queued request replay
